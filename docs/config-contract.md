# Configuration Notes

SignalForge Preview is JTL-first.

Most users should start here:

```powershell
signalforge run results.jtl
```

Optional configuration can help when you want:

- reusable environment metadata
- a named workload profile
- a deliberate APDEX override

Public preview guidance:

- start with the JTL alone
- add config only when it improves clarity
- treat advanced overrides as expert-only

Example files are available under `examples/config/`.
