# Feedback

The most useful preview feedback is practical and honest. The single most
important thing we want to learn: does SignalForge reduce the mental effort of
reviewing a run?

## Core feedback (all users)

1. Did SignalForge reduce the mental effort of reviewing this run compared to
   your usual process (raw JTL, spreadsheets, or other tooling)?
2. Was anything in the report unnecessary, or noise you would remove?
3. Was anything missing that you expected or needed?

These three matter most. Open, honest answers are more useful than polite ones —
including "it did not help me" or "I would not use this."

## Practical feedback

- Did install work cleanly?
- How long did it take to reach the first useful insight?
- What part of the report felt most helpful?
- Did compare feel meaningfully different from a generic HTML export?

## Additional feedback (Dynatrace users)

If you configured the optional, experimental Dynatrace enrichment:

4. Did you notice the Evidence Relationships section?
5. Did the categories (Agreement, No Corroboration, Partial Agreement,
   Insufficient Evidence) make sense without reading the docs?
6. Did it change how you investigated the run?
7. Did you trust the relationships shown, or did you still need to verify them
   manually in Dynatrace?
